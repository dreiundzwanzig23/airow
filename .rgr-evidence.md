Physical or interpretive claim:
- Users can export the reduced visualization artifact to a deterministic
  ParaView/VTK-compatible offline bundle that preserves state geometry, selected
  vector channels, units, frames, and provenance without changing runtime
  physics or claiming high-fidelity surfaces.

Fidelity level:
- reduced visualization.

Validity range:
- speed: inherited from emitted provider trust metadata.
- stroke rate: inherited from emitted scenario/configuration.
- wind/current/wave assumptions: exported only for vectors already present in
  `airow.visualization.v1`; unavailable disturbances remain unavailable.
- geometry assumptions: reduced hull/oar/blade reference geometry from emitted
  transforms, not production mesh surfaces.
- solver/timestep assumptions: export sample policy matches the source
  visualization artifact.

Oracle:
- visual invariant and deterministic file contract.
- details: the export must validate the source schema, emit stable legacy ASCII
  VTK `POLYDATA` files, and preserve vector unit/frame/provenance labels in a
  sidecar rather than embedding ambiguous names only.

Required outputs:
- time-series channels: unchanged.
- units: preserved in export metadata.
- frames: source `world`, `hull_body`, waterline, and vector frames.
- visualization vectors: all emitted sample vector channels as VTK point-data
  vectors when present.
- diagnostics: invalid visualization artifacts fail before export; report
  metadata lists generated ParaView files when requested.

Tests:
- UT: existing `D-063` artifact serializer coverage.
- IT: existing visualization artifact integration.
- QT: `QT-055`.

Failure modes:
- rejected: unsupported or malformed visualization schema.
- low-confidence: trust-envelope metadata remains in the source artifact/report.
- visually inspectable: unavailable channels are absent from vectors but remain
  documented in source artifact/report controls.
